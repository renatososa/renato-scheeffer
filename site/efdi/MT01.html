Módulo Técnico 1

Uno de los pilares fundamentales de un proyecto es la correcta documentación de todos los procesos que se desarrollan. En la EFDI (Especialización en Fabricación Digital e Innovación) el entorno de documentación que se propone es una página web pública, utilizando el ecosistema de GitHub. Para poder trabajar sobre la página web y generar al documentación del proyecto es necesario generar en nuestra PC un entorno de trabajo con todas las herramientas instaladas.

Instalación de las herramientas de trabajo: git

El primer software a instalar para trabajar en este módulo es git, este software compone un sistema de gestión de versiones el cual nos permite manejar las distintas modificaciones que vamos a realizar sobre nuestro repositorio (documentación) en el transcurso de la especialización.

Una vez descargado el software desde la página oficial de git procedemos a la instalación, en la Figura 1 se puede observar el ejecutable de instalación. Una vez presionamos instalar se comienzan a instalar todos los archivos.

Una vez instalado podremos abrir el software Git Bash el cual funciona cómo una terminal y nos permite trabajar con git desde la PC. Una forma de verificar que podemos trabajar con git correctamente es pararnos dentro de cualquier repositorio y ejecutar el comando git status como se puede observar en la Figura 2.

Con esto instalado utilizando comandos específicos ya podríamos manejar las versiones de nuestra documentación, sin embargo, existen interfaces gráficas que facilitan este trabajo y nos evitan el uso de comandos específicos, esto es lo que veremos en el siguiente apartado.

Instalación de las herramientas de trabajo: GitHub Desktop

GitHub Desktop es un software que nos permite trabajar con git de forma gráfica y sencilla. Utilizando botones nos permite realizar la mayoría de las acciones que podemos hacer con los comandos. En la Figura 3 se puede observar el proceso de instalación de GitHub Desktop una vez descargado desde la página oficial

Una vez instalado el software podremos abrirlo y comenzar a trabajar con un repositorio, desde el menu File -> Clone Repository accedemos al menú para clonar un repositorio de github ya existente. En la Figura 4 se puede observar la ventana de clonación. Es importante que seleccionemos correctamente el directorio donde queremos que quede clonado localmente el repositorio en la opción local path.

Instalación de las herramientas de trabajo: Visual Studio Code

Dado que el medio para documentar que vamos a utilizar es una página web para poder modificar el contenido de la misma es necesario que modifiquemos los códigos que la generan, para esto existen diversos softwares que nos permiten manipular código de forma un poco más sencilla, uno de ellos es Visual Studio Code (VSC). Uno de las principales fortaleces de VSC es su apartado visual que permite trabajar en un proyecto compuesto por muchos códigos de forma sencilla y reconociendo palabras clave de cada lenguaje de programación. También permite la integración de extensiones diseñadas para fines específicos, como por ejemplo, colorear el indentado.

En la Figura 5 se puede observar el ejecutable de instalación de VSC descargado de la página oficial.

Una vez instalado el software podremos ejecutar el programa y yendo a Archivo -> Abrir carpeta.. podemos abrir la carpeta de un proyecto como se observa en la Figura 6. En el panel derecho tenemos el explorador de archivos que nos permite movernos entre los archivos del proyecto de forma sencilla y rápida.

Creación de la página web con el template

Para generar la documentación los docentes de la EFDI nos proporcionaron un template el cual podríamos modificar para cargar la información de nuestro proyecto. Para poder trabajar sobre este template es necesario que generemos una "copia" del mismo en nuestro repositorio online de GitHub. Para "copiar" el repositorio debemos hacer Fork del repositorio template siendo el mismo fablabbcn.github.io/efdi-template/. Luego de presionar sobre Fork en el repositorio original nos aparece la ventana de la Figura 7. Luego de que presionamos Create fork tendremos una copia del template en nuestro repositorio online en mi caso en github.com/renatososa/renato-sosa

Luego de que tenemos el repositorio "copiado" para poder editarlo desde nuestra PC debemos generar una clave SSH y guardarla en el repositorio para que este se asegure de que nuestra PC tienen los permisos necesarios. Para generar la clave ssh debemos ejecutar en la consola de comandos lo siguiente: ssh-keygen -t rsa -b 4096 -C "your_email@example.com", lo que se obtiene luego de ejecutar el comando se puede observar en la Figura 8, esto nos asegura que la clave se generó correctamente.

Una vez tenemos la clave SSH creada debemos compartirla con el repositorio de GitHub, para esto debemos copiar el contenido del archivo ssh.pub generado y dirigirnos dentro del repositorio del template vamos a Settings -> Deploy Keys -> Add deploy key y pegamos el contenido de ssh.pub. Esta última ventana se puedo observar en la Figura 9 y luego de darle a Add key ya quedaría listo.

Ahora ya podremos generar una copia local en nuestra PC del repositorio que esta online en nuestra cuenta de GitHub. Para esto realizamos el procedimiento que se mencionó anteriormente utilizando la interfaz gráfica de GitHub Desktop, vamos a File -> Clone repository seleccionamos nuestro repositorio en mi caso renatososa/renato-sosa y le damos a Clone. Una vez se complete el proceso de clonado deberíamos ver la ventana de la Figura 10.

Luego de clonado tendremos en nuestra PC en el directorio que seleccionamos una copia de todos los archivos del repositorio y ya podremos editarlos utilizando VSC. Luego de realizar los primeros cambios sobre el template para que estos se vean reflejados en el repositorio online debemos hacer un commit y un push de estos cambios desde GitHub Desktop

Una vez realizado el primer commit en nuestro repositorio online de GitHub se comienzan a ejecutar las acciones estipuladas en el archivo main.yml donde en resumen, se genera una rama gh-pages y en la misma se hace el build de la página web. En la Figura 11 se puede observar el Work Flow generado a partir del primer Commit realizado. El tic en verde nos indica que todas las acciones se realizaron correctamente.

Una vez que las acciones fueron ejecutadas debemos configurar en GitHub la página para que sea visible públicamente y se pueda acceder mediante un enlace. Para esto dentro de repositorio vamos a Settings -> Pages y en Branches seleccionamos gh-pages. Con esto le decimos que la página web será cargada desde la rama gh-pages. En la Figura 12 se observa cómo debería quedar esta configuración. Presionando en save guardamos los cambios y GitHub automaticamente nos genera el link para visualizar nuestra página.

En mi caso el link generado es https://renatososa.github.io/renato-sosa/ en la Figura 13 se puede observar mi template de la página web del EFDI ya con algunos de mis datos cargados.
